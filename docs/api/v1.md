# Context API v1 (Lab / BFF)

Status: **frozen** for additive evolution ([ADR-0026](decisions/0026-public-api-v1-freeze.md)).  
Transport: HTTP+JSON (`cmd/context-serve`) and Go client (`pkg/contextkit`).

`api_version` = `v1`. Servers set header `X-Context-API-Version: v1`.

## Graph & query semantics (stabilization S4)

- Graph storage/traversal is **forever-deferred**; consumers own edge
  projections ([ADR-0040](decisions/0040-graph-consumer-projection.md)).
- No Query AST in core; power-user path = modes + `RetrievalFilters` + Postgres
  FTS ([ADR-0041](decisions/0041-query-ast-defer-fts-filters.md),
  [search-power-user.md](../search-power-user.md)).

## Adapters (stabilization S3)

- Public language harness: `pkg/langtestkit` + `pkg/langcontract`
  ([ADR-0037](decisions/0037-public-langtestkit.md)).
- Thin adapters: `context-lang-en`, curated JSON lexicon, HTML/PDF parsers
  (confidence on extract), NDJSON event source
  ([ADR-0038](decisions/0038-s3-thin-adapters.md)).
- Freeze-defer: DOCX, object-store artifacts, fuzzy/`pg_trgm`; HTTP models are
  the supported production Completer/Embedder path
  ([ADR-0039](decisions/0039-s3-adapter-freeze-defer.md)).

## Search snippets & tool approval (stabilization S2)

- `POST /v1/search` candidates may include additive `snippet` (`text`,
  `chunk_span`, `highlights`, `chunk_checksum`) — absolute offsets into chunk
  text ([ADR-0033](decisions/0033-offset-stable-snippets.md)).
- Write/external tools without an allow rule decide `ask`; agent-run records
  tool status `needs_approval` and does not execute
  ([ADR-0034](decisions/0034-tool-side-effect-approval.md)).
- Prompt-injection fixtures: `go test ./internal/evals/adversarial/`
  ([ADR-0035](decisions/0035-prompt-injection-fixtures.md)).
- Post-merge reranker hook is wired (`Identity` on CLI search)
  ([ADR-0036](decisions/0036-intentional-reranker-path.md)).

## Soft quotas (Chunk 28)

Env (0/unset = unlimited): `CONTEXT_QUOTA_MAX_CHUNKS`, `CONTEXT_QUOTA_MAX_PACKS`,
`CONTEXT_QUOTA_MAX_RUNS`, optional `CONTEXT_QUOTA_SOFT_ASK_PERCENT` (default 80).

- Soft threshold → decision `ask` (advisory; writes still allowed).
- Hard limit (`used >= max`) → decision `deny`; ingest / context-pack / agent-run
  return permission error (HTTP 403). No billing.

## Index lifecycle (stabilization C1 / ADR-0032)

`GET /v1/index?project_id=` returns `phase` (`empty|ready|degraded|rebuilding|failed`),
`search_available`, and stable `reasons`. Rebuild keeps the active ready
snapshot searchable (`rebuild_in_progress` does not flip active).

## Failure / degraded (Chunk 29)

- Missing dense/sparse retriever on a requested strategy → `unavailable` (not empty hits).
- Hybrid + `CONTEXT_ENABLE_DENSE` when vector fails → `200` with `degraded` + reasons
  (exact/sparse still run). Modes `dense` / `hybrid-dense` fail hard.
- Failure injection: `CONTEXT_FAIL_METADATA|VECTOR|SPARSE|EMBEDDER|ARTIFACT|COMPLETER=1`.
- `/health` stays live (`ok:true`) and embeds readiness; `/v1/ready` is `503` when not ready.

## Redaction (Chunk 30)

- `CONTEXT_REDACT` defaults **on** (`0`/`false`/`off` to disable for local debug).
- Applies to agent-run `model_text` and inspect `surface_preview` only (corpus/index untouched).
- Patterns: bearer tokens, api_key/password/token assignments, email-like addresses.
- Response fields: `redacted`, optional `redact_count`.

## Source tombstones (stabilization C1 / ADR-0028)

`POST /v1/sources/tombstone` with `{project_id, source_id}` soft-deletes a
source. Chunks remain on disk but leave search and new packs. Re-ingest of the
same source clears the tombstone. Additive within API v1.

## Snapshot bundles (stabilization C2 / ADR-0029)

- `POST /v1/snapshot/export` `{project_id}` → sealed `bundle` + checksum.
- `POST /v1/snapshot/import` `{project_id, activate, bundle}` → verify first;
  `activate:true` flips active snapshot only when verify passes. Corrupt/partial
  bundles never become active. Host paths are not in the bundle.

## Project export / delete (stabilization C7 / ADR-0030)

- `POST /v1/project/export` `{project_id, out_path}` → `project-archive-v1`
  (snapshot bundle + focuses; no host paths).
- `POST /v1/project/delete` `{project_id, confirm_project_id}` → tombstones
  sources, deletes metadata/artifacts, clears workspace state. Confirm must
  equal `project_id`.

## Background jobs (Chunk 31)

In-process execution. Kind: background `AgentRun`.

| Method | Path | Notes |
| --- | --- | --- |
| POST | `/v1/jobs` | `{project_id,query,owner,focus_id?}` → `202` |
| GET | `/v1/jobs` | list |
| GET | `/v1/jobs/{id}` | status |
| POST | `/v1/jobs/{id}/cancel` | cancel pending/running |

`owner` required. Mid-flight jobs die if the process exits.

## Schedules (stabilization C8 / ADR-0031)

Durable triggers under `ops/schedules`. Kinds: `once_at`, `interval`, `event`.
Set `enabled: true`. After restart, `POST /v1/schedules/tick` (also run once on
`context-serve` start) enqueues overdue schedules as **new** jobs.

| Method | Path | Notes |
| --- | --- | --- |
| PUT | `/v1/schedules` | upsert schedule JSON |
| GET | `/v1/schedules?project_id=` | list |
| DELETE | `/v1/schedules/{id}?project_id=` | delete |
| POST | `/v1/schedules/tick` | fire due `once_at` / `interval` |
| POST | `/v1/schedules/fire` | `{project_id,event_type}` |

## Compatibility

- Documented JSON fields: do not rename/remove within v1.
- Optional new fields and routes are allowed.
- `project_id` mismatch → permission error (HTTP 403), not silent widen
  ([ADR-0025](decisions/0025-multi-tenant-isolation.md)).
